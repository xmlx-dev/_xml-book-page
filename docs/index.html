    XML Book Documentation¶

    Welcome to the XML book documentation. If you are looking for the book itself, navigate to book.xmlx.io.

    Repository Structure¶

    The book has two main components:

    • content authored in Jupyter Book and published at book.xmlx.io; and
    • documentation built with MkDocs and accessible at book.xmlx.io/docs.

    The book source is stored in the book directory and the documentation source is kept in the docs directory of the xmlx-io/xml-book GitHub repository. We keep the documentation separate to ensure a clean structure and immersive experience when reading the book.

    About the Documentation¶

    This documentation covers technical concepts to do with how the book is built and maintained, as well as a contributor’s guide that outlines how to author new content for the book. In particular, you will find here a code of conduct, changelog, ways to contact us and acknowledgements. The documentation also includes a dedicated page describing our preferred contribution workflow and information how to set up your development environment. There are specific pages for working with the documentation and the book.

    Reporting an Issue¶

    If you find a typographical error or a conceptual mistake in the book, please open a GitHub issue to inform us about it – there are a number of predefined templates to help you with reporting. If the problem requires more effort than just a few simple text edits and you are happy to fix it yourself, please visit the development page to get started.
